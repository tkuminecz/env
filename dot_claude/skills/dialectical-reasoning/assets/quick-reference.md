# Dialectical Reasoning Quick Reference

## Phases at a Glance

| Phase | Goal | Hard Stop? |
|-------|------|-----------|
| 1 — Elenctic Interview | Surface the real question & belief burden | After framing confirmed |
| 2 — Thesis | Strongest version of Position A | — |
| 3 — Antithesis | Strongest version of Position B (genuine negation) | — |
| 4 — Determinate Negation | Specific failure modes; Boydian decomposition | ✅ Present before Phase 5 |
| 5 — Synthesis | Aufhebung: cancel, preserve, elevate | Present before Phase 6 |
| 6 — Validation | Adversarial check; hostile auditor | — |
| 7 — Recursion | Synthesis → new thesis; recurse if needed | — |

## Synthesis Pattern Decision Tree

```
Is the tension about TIMING?
  └─ Yes → Temporal Synthesis (Phase A then Phase B)

Is the tension about CONTEXT/CONDITIONS?
  └─ Yes → Conditional Synthesis (A when X, B when Y)

Are the goals actually on DIFFERENT AXES?
  └─ Yes → Dimensional Synthesis (optimize A and B orthogonally)

Are both positions TACTICS for the same deeper goal?
  └─ Yes → Higher-Order Synthesis (find better tactic or reframe goal)

Is one position correct but RISKY?
  └─ Yes → Compensating Controls (A + B as guardrail)

Is the QUESTION itself wrong?
  └─ Yes → Reframed Stakes (dissolve the framing, answer the right question)
```

## Anti-Sycophancy Checks

Before delivering synthesis:
- [ ] Could either side have proposed this without the dialectical process?
- [ ] Is this "a bit of A + a bit of B" (weighted average, not synthesis)?
- [ ] Does the synthesis locate the user's implicit preference and converge on it?
- [ ] Can every synthesis claim be traced to the Phase 4 atomic decomposition?
- [ ] Is the synthesis more abstract than both positions in a way that actually *says something*?

## Quick-Start Mode (Low Stakes / Time Constrained)

```
Q1 — Frame: establish thesis and antithesis in 2-3 exchanges
Q2 — Steelman: apply Phase 2 and Phase 3 rigor in one pass each
Q3 — Synthesize: apply Phase 5 directly, skip validation
```

Signal: user says "quickly," "rough sketch," "brief," or low-stakes domain.
